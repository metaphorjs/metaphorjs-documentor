Methods

  • pcall (name)  -> *

    Call a plugin function and pass parameters to it. Plugins are js files from hooks directory containing functions, scalar values and any other stuff. The file structure is like file/js/item/param/add.js which is called when adding parameter to js function. Plugins can be generic: file/* /item/* /* /add.js . Plugins are stored in globalCache and can be overriden/intercepted etc.

    Internally, it uses pget to find the plugin.

    Called from SourceFile it looks for file/-ext-/-name- plugins. Called from Item it looks for file/-item-file-ext-/item/-current-item-type-/-name- and if not found - generics. First item_type generic, then file_ext generic.

    Also, there is '?' plugin directories for 'other' functions.

    • string name

      Plugin name

  • pget (name, collect, passthru, exact, merge)  -> *

    Find an applicable plugin(s). Plugins are not only functions, but also scalar values and objects.

    • string name

      Plugin name

    • boolean collect
      • @default false

      Collect all applicable plugins: including exactly matched plugin and all generics. Returns array

    • function passthru ()  -> boolean

      Call this function with each found plugin. If returned false, execution stops and last plugin returned. Does not work with collect.

      • * value
    • boolean exact
      • @default false

      Only return the first matched plugin, if any

    • boolean merge
      • @default false

      Not only collects (requires collect=true), but also merges all returned objects into one. Returns object
